# Минимальный мониторинг перед продом

## На что смотреть (алерты)

1. **Контейнеры не в состоянии running** — один из сервисов (postgres, n8n, userbot, editor-bot, nginx) упал или перезапускается.
2. **Рост ошибок webhook** — в логах userbot частые `webhook_all_retries_failed` или `webhook_failed`; значит n8n или сеть недоступны.
3. **Диск заполнен (>80%)** — риск падения БД и записи PDF. Следить за разделом, где лежат `postgres_data` и `pdf_storage`.

## Простая проверка контейнеров (cron)

Скрипт [../scripts/check_services.sh](../scripts/check_services.sh) проверяет, что все контейнеры проекта в состоянии `running`. При необходимости настройте cron с отправкой вывода по почте или в Telegram (например через бота или `mail`):

```bash
# Каждые 5 минут проверка; при ошибке — вывод в stderr (cron отправит на MAILTO=)
*/5 * * * * cd /home/goutach/parser && ./scripts/check_services.sh
```

Без внешнего мониторинга этого достаточно для быстрого обнаружения падения сервисов.
